Details

  • The lego arena will be of 250*250 cm (5*5 blocks)
  • The lego robot will have to navigate around the arena to find path and locate targets and pot balls.
  • There will be only one possible open path and the entrance and exit will be variable.
  • There will be markers marking the exit.
  • Your brick can only have a single program on it at the time of the competition.
  • The robot at each time will have only one open path where it has to go.
  • The path will be random and will be defined at the start of the match.
  • The robot will have to find the path on it’s own.
  • The robot will be able to distinguish between the entry gate and exit gate. The entry gate will have a red box placed on its right side. The exit gate will have a blue box placed on its right side.
  • For the entry gate, right is with respect to the robot’s orientation when it is facing the arena standing at the start point
  • For the exit gate, right is with respect to the robot’s orientation when it is inside the arena facing the exit.
    •

    Mechanism

  • The robot will have to pot balls of 2 different colors into corresponding boxes.
  • The balls will be preloaded on the robot.

    Guidlines for the proposal

    Your proposal should include the following:
    1. An Overview which explains the theme in your own words or what you understand from it.
    2. Tools and References which includes the tools you will use for example lego mindstorms nxt, the softwares (nxt gui software and robotC software), the sensors you will need, the websites you want to refer to or think will be useful.
    3. Methodology which will describe how you plan to acheive the following tasks:
      • Wall Following where you make the robot follow a wall. Note: Please list which sensors you will use to do this and a rough idea of how this sensor works
      • Color Sensing where you make the robot sense the color of a box in the maze. Note: Please list which sensors you will use to do this task and how this sensor works
      • Ball Potting where you will describe what sort of mechanism you will use to pot the balls preloaded on the robot. Note:Take the time to research lego robots that do this. You may look for foraging robots, pick and place kind of robots and so on. Look at the mechanisms and find out which ones you'd like to use
      • Maze Solving where you describe some idea of how the robot can solve the maze. Note: The robot doesn't have to be super intelligent. It just needs to know when to get out of the maze and when to pot the ball.
    4. Work Distribution and Timeline where you follow the timeline described above and describe in more detail what you think you will be doing for the duration of the project. For example before issuing kits for module 2 again, you may work on learning RobotC and completing the software part.
    5. Testing and Debugging where you describe in some detail how you think you can test your robot after each module. For example for wall following you may test the robot in the mts corridor. You may first test the output of the sensors you use, or test that you can make the robot move and so on.
